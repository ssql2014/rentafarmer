产品特色

真实地块、真实农民、全程可看

有机方式管护

默认不打农药，优先采用人工除草、物理防虫、生物防治和农事隔离。若用户需要认证级有机产品，需由接口人核验证书。

24 小时视频监控

示范地块接入实时视频，用户可以远程查看播种、浇水、长势、采摘和现场服务过程。

当地农民服务

由当地农民负责浇水、除草、巡田、采摘、打包和日常维护，区域接口人负责调度和验收。

成熟后灵活处理

瓜果成熟后，用户可以自行到地块采摘，也可以选择邮寄到家，或委托接口人代为销售。

安装与调用

CLI、Codex Skill、Claude Skill

1. 获取项目

先 clone 公共仓库，后续 CLI、Codex 和 Claude 都复用这份目录。

git clone https://github.com/ssql2014/rentafarmer.git
cd rentafarmer
npm test

2. 安装 CLI

本机注册 `rentafarmer` 命令，用自然语言生成种植建议。

npm link
rentafarmer recommend "我想种番茄和黄瓜，地块 2 分，全日照，沙壤土"

3. 安装到 Codex

把仓库作为 Codex skill 目录挂到本机 Codex skills 下。

mkdir -p ~/.codex/skills
ln -s "$PWD" ~/.codex/skills/rentafarmer

4. 安装到 Claude

Claude 同样可以通过 skills 目录读取根目录的 `SKILL.md`。

mkdir -p ~/.claude/skills
ln -s "$PWD" ~/.claude/skills/rentafarmer

OpenClaw 安装调试

把 RentAFarmer 接入长期运行的代理入口

1. 安装 OpenClaw

通过 npm 全局安装 CLI，然后初始化本机配置。

npm install -g openclaw
openclaw --version
openclaw setup
openclaw configure

2. 安装 RentAFarmer Skill

把当前公开仓库挂到 OpenClaw/Codex 可读取的 skills 目录。

git clone https://github.com/ssql2014/rentafarmer.git
mkdir -p ~/.codex/skills
ln -s "$PWD/rentafarmer" ~/.codex/skills/rentafarmer
openclaw skills list

3. 启动与健康检查

检查网关、频道、模型和本机配置是否可用。

openclaw config validate
openclaw doctor
openclaw status
openclaw health

4. 调试日志

出现消息不转发、技能不可见或网关异常时，先看这些信息。

openclaw logs
tail -f ~/.openclaw/logs/gateway.log
tail -f ~/.openclaw/logs/gateway.err.log
openclaw channels list
调试建议 先跑 `openclaw doctor`，再看 `openclaw status` 和 gateway 日志。不要把微信、Discord、模型密钥写进网页或仓库。

底层能力

把 AI 作为种植顾问入口

平台对用户提供自然语言入口，先输出标准 JSON：意向、地块条件、推荐品种、面积、费用和微信承接话术。 后续由人工确认地块、农时、视频监控、当地农民服务、成熟后采摘/邮寄/代卖和最终报价。 

POST /api/plan
{
  "text": "我想种番茄和黄瓜，地块 2 分，全日照，沙壤土"
}

边界

先咨询，再人工确认

  • AI 输出是种植意向建议，不构成最终报价。
  • 确认意向后加微信，由人工核地块、农时、配送和合同边界。
  • 服务特色为有机方式管护、不打农药和 24 小时视频监控；如需“有机认证”，需另行核验证书。
  • 瓜果成熟后可自行采摘、邮寄，或委托接口人代为销售。
  • 认养按农事服务或消费权益处理，不承诺保本、固定收益或回购。
  • 高风险作业由认证主体承接，村级协理只做复核、见证和调解。
